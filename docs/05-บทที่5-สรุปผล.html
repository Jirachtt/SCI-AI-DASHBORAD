บทที่ 5
สรุปผลและข้อเสนอแนะ

5.1 สรุปผลการดำเนินงาน

จากการพัฒนาโครงงาน "Science AI Dashboard: ระบบ Dashboard อัจฉริยะสำหรับคณะวิทยาศาสตร์ มหาวิทยาลัยแม่โจ้ โดยใช้เทคโนโลยี AI" สามารถสรุปผลได้ดังนี้

  1. ด้านการพัฒนาระบบ: สามารถพัฒนาระบบ Web Application ด้วย React 19 ร่วมกับ Vite 7 ที่ครอบคลุมข้อมูล 6 หมวดหลัก ได้แก่ บุคลากร (HR), นักศึกษา (Student), งานวิจัย (Research), การเงิน (Finance), ยุทธศาสตร์ (OKR), และ AI Chatbot รวมทั้งสิ้น 15 หน้าจอ ทั้งหมดสามารถแสดงผลข้อมูลในรูปแบบกราฟ ตาราง และ Card ที่อ่านเข้าใจง่าย
  2. ด้าน AI Chatbot: สามารถพัฒนา AI Chatbot ที่ขับเคลื่อนด้วย Google Gemini 2.5 Flash API ได้สำเร็จ โดย AI สามารถตอบคำถามเกี่ยวกับข้อมูลมหาวิทยาลัยเป็นภาษาไทย, สร้างกราฟ (Bar, Line, Pie, Radar) อัตโนมัติ, พยากรณ์แนวโน้มข้อมูล, ค้นหานักศึกษา และรับคำสั่งด้วยเสียง
  3. ด้าน Data Visualization: สามารถแสดงผลข้อมูลผ่านกราฟ 7 ประเภท (Bar, Line, Doughnut, Pie, Radar, Polar Area, Stacked Bar) โดยใช้ Chart.js พร้อมฟีเจอร์ Zoom และ Pan ที่ช่วยให้ผู้ใช้สำรวจข้อมูลเชิงลึกได้
  4. ด้าน Role-Based Access Control: สามารถพัฒนาระบบควบคุมสิทธิ์ 4 ระดับ (Dean, Lecturer, Staff, Student) ที่ทำงานร่วมกับ Firebase Authentication ได้อย่างมีประสิทธิภาพ
  5. ด้าน Responsive Design: ระบบรองรับการใช้งานบนอุปกรณ์หลากหลายขนาดหน้าจอ ทั้ง Desktop, Tablet และ Mobile โดยใช้ CSS Grid, Flexbox และ Media Queries

5.2 ปัญหาและอุปสรรค

  1. API Rate Limit: Google Gemini API เวอร์ชันฟรี (Free Tier) มีข้อจำกัดด้านจำนวน Request ต่อนาที ทำให้ AI Chatbot อาจไม่สามารถตอบสนองได้ทันทีเมื่อใช้งานหนัก ได้แก้ไขโดยเพิ่มระบบ Auto-Retry ด้วย Exponential Backoff
  2. โมเดล AI ที่เปลี่ยนแปลง: Google มีการเปลี่ยนแปลงชื่อโมเดล AI อย่างต่อเนื่อง โมเดลบางตัว (เช่น gemini-pro, gemini-1.5-flash) ถูกยกเลิกไป ต้องอัปเดต Model Fallback Chain อยู่เสมอ
  3. ข้อมูลจำลอง: เนื่องจากไม่สามารถเข้าถึงฐานข้อมูลจริงของมหาวิทยาลัยได้ จึงต้องสร้างข้อมูลจำลอง (Mock Data) ที่สมจริง ซึ่งอาจไม่ตรงกับข้อมูลจริงทุกประการ
  4. ขนาดของ System Instruction: การส่งข้อมูล Dashboard ทั้งหมดไปเป็น System Instruction ของ AI ทำให้ Request มีขนาดใหญ่ ซึ่งอาจส่งผลต่อความเร็วในการตอบสนองและค่าใช้จ่าย Token
  5. การรองรับภาษาไทยในเสียง: Web Speech API มีความแม่นยำในการรับรู้เสียงภาษาไทยไม่สูงมาก โดยเฉพาะคำศัพท์เฉพาะทาง

5.3 ข้อเสนอแนะ

5.3.1 ข้อเสนอแนะสำหรับการพัฒนาต่อ

  1. เชื่อมต่อฐานข้อมูลจริง: เชื่อมต่อกับ API ของระบบทะเบียน ระบบ ERP หรือ Data Warehouse ของมหาวิทยาลัย เพื่อแสดงข้อมูลที่เป็นปัจจุบัน (Real-time Data)
  2. เพิ่ม Backend Server: พัฒนา Backend ด้วย NestJS หรือ Express.js เพื่อจัดการ API Key ฝั่ง Server (ปลอดภัยกว่าฝั่ง Client) และรองรับ Caching, Rate Limiting
  3. พัฒนาระบบรายงาน (Report Generation): เพิ่มฟีเจอร์ Export เป็น PDF หรือ Excel สำหรับรายงานประจำเดือน/ปี
  4. พัฒนาระบบแจ้งเตือน (Notification): แจ้งเตือนผู้บริหารเมื่อมี KPI ต่ำกว่าเป้า หรือข้อมูลผิดปกติ
  5. เพิ่ม Machine Learning Models: พัฒนาโมเดลพยากรณ์เฉพาะทาง เช่น Drop-out Prediction, Budget Optimization
  6. รองรับ Multi-Language: เพิ่มภาษาอังกฤษสำหรับผู้ใช้ต่างประเทศ

5.3.2 ข้อเสนอแนะสำหรับนักศึกษาที่สนใจ

  1. ศึกษา React Hooks อย่างลึกซึ้ง โดยเฉพาะ useState, useEffect, useCallback เพราะเป็นพื้นฐานของ Modern React
  2. เรียนรู้การใช้ AI API (เช่น Gemini, GPT) ในการพัฒนา Application เพราะเป็นทักษะที่มีความต้องการสูงในตลาดแรงงาน
  3. ให้ความสำคัญกับ UI/UX Design เพราะ Dashboard ที่สวยงามและใช้งานง่ายจะช่วยเพิ่มคุณค่าให้โครงงาน
  4. ใช้ Version Control (Git) ตั้งแต่เริ่มโครงงาน เพื่อติดตามการเปลี่ยนแปลงและสามารถย้อนกลับได้เมื่อเกิดปัญหา
